# ERP 程式撰寫安全規則 (Secure Coding Guidelines)

為了確保 ERP 系統的資料安全性與防禦網路攻擊，所有開發工作必須嚴格遵守以下安全規範：

## 1. 徹底防範 SQL 注入 (SQL Injection)
*   **禁止使用動態 SQL 拼接**：嚴禁在程式碼中直接使用字串拼接組合 SQL 語法。必須透過 `database/queries.js` 預定義查詢，並使用 `NamedQuery ($1, $2)` 機制將資料存取抽象化。
*   **伺服器端強制過濾**：
    *   **不得僅依靠前端 Javascript 進行檢查**，以避免駭客繞過瀏覽器直接發送命令。
    *   所有輸入參數必須嚴格過濾下列字元彙總：`|`, `&`, `;`, `$`, `%`, `@`, `'`, `"`, `\'`, `\"`, `( )`, `+`, `CR (0x0d)`, `LF (0x0a)`, `,`, `\`。
    *   **Mass SQL 關鍵字過濾**：必須濾除 `Select`, `Insert`, `Dbo`, `Declare`, `Cast`, `Drop`, `Union`, `EXEC`, `NVARCHAR` 等關鍵字，以防範大規模 SQL 攻擊。

## 2. 資訊洩漏與日誌防護
*   **隱藏系統資訊**：嚴禁在畫面上輸出任何日誌紀錄（Log）或除錯資訊（Debug Log）。
*   **標準化錯誤回傳**：應避免將 Web 平台（如 IIS/Apache）或程式語言的預設系統錯誤碼直接顯示給使用者。API 應回傳標準化的通用訊息。
*   **資料存取抽象化**：避免使用者直接存取 Table 或 View，應透過模組化接口執行資料交換。

## 3. 資料保護與連線安全
*   **敏感資訊處理**：重要資訊（如使用者密碼）嚴禁以「明文」儲存。必須使用雜湊（Hash）或加密機制處理，且金鑰不得硬編碼於程式碼中。
*   **資料殘存防範**：應用程式處理資料後，應避免產生資料殘存（如暫存檔或留存於記憶體中），所有暫存資源應於程式執行完畢後立即刪除。
*   **資料庫存取限制**：資料庫嚴禁被未經授權的外部 Script 程式直接存取。

## 4. 檔案系統與執行安全
*   **上傳資料夾安全**：嚴禁在存放上傳檔案的資料夾內執行任何動態網頁程式（如 .php, .aspx, .js 等）。
*   **禁止讀取系統檔案**：程式開發需嚴格限制路徑權限，避免利用 Web 應用程式直接讀取系統底層檔案（如 /etc/passwd 或 Windows 系統檔）。

## 5. 前端防護 (XSS)
*   **React 安全機制**：利用 React 預設的轉義（Escaping）特性顯示資料，避免使用 `dangerouslySetInnerHTML`。
*   **媒體路徑過濾**：所有圖片與檔案連結應透過統一的轉換器（如 `window.getMediaUrl`），防範惡意路徑指向系統目錄。

---
**開發備註**：如涉及任何資料庫結構變動，請務必先更新 `queries.js` 及其對應的參數過濾邏輯。
